# YuanxiaoMonitor Bot 访问策略

感谢您查看 YuanxiaoMonitor 的访问策略。本页面旨在向网站管理员清晰说明我们的爬虫（Bot）的行为规范。

## 关于 YuanxiaoMonitor

YuanxiaoMonitor 是一个**院校官网更新监测工具**。它通过定期检查指定网页的内容变更，帮助用户（如学生、关注者）及时获取目标院校的最新通知。

## 我们如何抓取

为确保不给目标服务器带来不必要的负担，我们严格遵守以下原则：

### 1. 遵循 robots.txt

我们的爬虫完全尊重您网站根目录下的 `robots.txt` 文件规则：

1.  **遵循 robots.txt**：我们的爬虫会完全尊重您网站根目录下的 `robots.txt` 文件规则，不会访问被禁止的路径。
2.  **控制访问频率**：
    *   robots.txt 缓存 1 小时，获取失败时默认允许访问（fail-open）。
    *   我们不会对您的服务器造成高并发压力，请求速率通常保持在一个较低的水平。****

### 2. 控制访问频率

我们对请求频率设置了多级保障，全部通过代码硬性约束，不可被用户配置绕过：

- **调度间隔**：定时检查的最小间隔硬性限制为 **10 分钟**，随机抖动 **±5 秒**（纳秒级种子），不可配置
- **爬虫延迟**：每次请求之间的最小间隔硬性限制为 **1500 毫秒**（可在线程设置页面调整，最低不低于 1500ms，默认 3000ms），同时遵循目标网站 `robots.txt` 中的 Crawl-Delay 指令，取两者较大值

### 3. 自动熔断保护

爬虫内置站点级熔断器（Circuit Breaker），在检测到目标服务器压力时主动挂起：

- 连续收到 **3 次 HTTP 503（Service Unavailable）或 429（Too Many Requests）** 后，该站点被自动挂起 **12 小时**
- 挂起期间不再对该站点发起任何请求，直接跳过检查并记录日志："目标服务器限制访问，已自动暂停(12h)"
- 只要出现 1 次正常响应（非 503/429），失败计数即重置为 0
- 12 小时后自动解除挂起，恢复正常检查

### 4. 探针健康度监控

爬虫持续跟踪每个站点的可访问状态：

- **正常**：站点响应正常，不产生额外通知
- **警告**：连续 2 次检查失败，面板显示黄色指示灯
- **严重**：连续 5 次检查失败或熔断触发，面板显示红色指示灯，并通过桌面通知和微信推送告警
- 站点恢复后自动推送恢复通知，不会重复发送严重告警

### 5. 身份标识

我们的每次请求都会携带明确的 User-Agent 标识，方便您识别和统计：

```
YuanxiaoMonitor/1.0 (院校官网监测; +https://github.com/claw-manager/yuanxiao-monitor)
```

该标识包含应用名称、版本、用途说明以及本策略页面的链接。

## 网站管理员指南

如果您希望我们的爬虫停止抓取您的网站，可按以下方式操作：

### 通过 robots.txt 控制

在您网站根目录的 `robots.txt` 文件中添加以下规则即可：

```txt
User-agent: YuanxiaoMonitor/1.0
Disallow: /
```

添加后，用户尝试将您的网站添加到监控时会收到错误提示："目标网站 robots.txt 禁止自动访问"。

### 通过响应状态码控制

如果我们的爬虫对您的服务器造成了压力，您可以通过返回 HTTP 503 或 429 状态码来触发熔断器。在连续 3 次收到此类响应后，爬虫会自动暂停对该站点的检查 12 小时。
